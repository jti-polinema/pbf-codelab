Mengapa Quasar Framework ?

Karena Quasar dapat menghemat biaya pengembangan perangkat lunak dengan sangat drastis. Hanya Quasar yang dapat melakukan ini, lihat video berikut:

Selain itu, Quasar merupakan:

  • Berbasis Vue.js
  • Mendukung pedoman UI Material untuk web dan app
  • Sangat baik mendukung aplikasi desktop dan mobile browser (termasuk iOS Safari)
  • Sangat mendukung pengembangan multiplatform seperti mode SPA, SSR, PWA, Mobile app, desktop app dan browser extension
  • Sangat mudah mengelola library CSS dan JS
  • Framework yang fokus terhadap performa sistem
  • Dan masih banyak kelebihan lainnya (baca lebih lanjut di sini)

Quasar merupakan solusi nomor satu berbasis Vue yang mendukung untuk membangun aplikasi desktop, web, mobile, atau bahkan semuanya sekaligus.

Apa itu Quasar ?

Quasar (baca /ˈkweɪ.zɑɹ/) adalah sebuah framework dengan lisensi open-source dari MIT yang berbasis Vue.js, mendukung seorang web developer untuk membuat web/app secara responsive++ dengan cepat dalam berbagai platform:

  • SPA (Single Page App)
  • SSR (Server-side Rendered App) (+opsional PWA client)
  • PWA (Progressive Web App)
  • BEX (Browser Extension)
  • Mobile apps (Android, iOS) dengan Cordova atau Capacitor
  • Multi-platform desktop apps (menggunakan Electron)

Moto Quasar adalah "menulis kode sekali dan secara simultan di deploy" sebagai sebuah web, mobile app dan/atau Electron App.

Ketika menggunakan Quasar, kita tidak perlu lagi menambah library seperti Hammer.js, Moment.js atau Bootstrap. Karena semua itu sudah tersedia di dalam Quasar.

Apa lagi yang didapat ?

Karena kemudahan dalam penggunaan dan powerfull, Quasar tersedia dalam paket full fitur, yang dibangun untuk kemudahan bagi developer. Beberapa fitur yang dimiliki oleh Quasar adalah sebagai berikut:

Semua Platform dalam Satu Portal

Satu sumber kode otoritatif untuk semua platform secara simultan: aplikasi web responsive untuk desktop/mobile (SPA, SSR+SPA, SSR+PWA), PWA, mobile app, dan multi-platform desktop app (Electron).

Paket Komponen Terbesar dan Terbaik di kelasnya, cepat, dan responsif

Ada sebuah komponen yang hampir setiap pengembangan web dibutuhkan dalam Quasar. Setiap komponen Quasar dibuat dengan hati-hati agar dapat memberikan pengalaman pengguna yang terbaik. Quasar dirancang dengan performa dan dukungan responsif yang sangat baik. Perhatian terhadap kinerja dan desain yang baik adalah sesuatu yang membuat bangga Quasar itu dibuat.

Praktik Terbaik Diintegrasikan Secara Default

Quasar juga dibangun untuk mendorong pengembang mengikuti praktik terbaik pengembangan web. Untuk melakukan ini, Quasar dikemas penuh dengan fitur-fitur hebat.

  • HTML/CSS/JS minification
  • Cache busting
  • Tree shaking
  • Source-mapping
  • Code-splitting with lazy loading
  • ES6 transpiling
  • Linting code
  • Accessibility features

Quasar menangani semua praktik terbaik pengembangan web dan lainnya - tanpa perlu konfigurasi.

Pada praktikum ini, kita mulai menggunakan Quasar Framework. Ada tiga cara untuk memulai menggunakan Quasar, yaitu:

  • Menggunakan UMD/Standalone (melakukan embed kode ke project yang sudah kita punya dengan link CDN, sehingga terintegrasi secara progresif)
  • Menggunakan Quasar CLI (Command Line Interface), cara kedua ini adalah yang paling direkomendasikan
  • Menggunakan plugin Vue CLI 3

Untuk membaca lebih lanjut terkait perbandingan ketiga cara tersebut, dapat membacanya di sini.

Kita akan memulai menggunakan Quasar Framework dengan cara kedua, yaitu Quasar CLI, silakan lakukan langkah-langkah praktikum berikut ini.

  1. Buka CMD atau terminal pada OS Anda, lalu ketik perintah berikut untuk install Quasar CLI.

Catatan: Sebaiknya yarn sudah terinstall pada komputer Anda. yarn merupakan package manager seperti npm, namun memiliki performa yang lebih baik. Baca lebih lanjut terkait NPM vs Yarn. Cara install yarn adalah sebagai berikut:

npm install -g yarn

yarn global add @quasar/cli
# atau
npm install -g @quasar/cli

Perhatian: Versi Node >= 10 dan npm >= 5 harus sudah terpasang pada komputer Anda. Jangan menggunakan Node > 14, karena Quasar masih menggunakan Webpack 4, belum diuji untuk versi yang lebih tinggi.

  1. Setelah proses instalasi Yarn dan Quasar berhasil, selanjutnya kita buat folder project Quasar baru pada lokasi dokumen sesuai keinginan kita dengan perintah berikut.
quasar create belajar_quasar
  1. Pada saat proses pembuatan project Quasar, maka akan tampil beberapa pertanyaan untuk kita konfigurasi pada project Quasar seperti berikut.

Perhatian: Perhatikan teks berwarna hijau tosca pada gambar di atas. Teks tersebut merupakan inputan dari kita. Mulai dari project name hingga akhir pemilihan package manager menggunakan yarn.

  1. Tunggu hingga proses pembuatan project folder sampai selesai. Jika prosesnya sudah selesai, maka akan tampil seperti pada gambar berikut.

  1. Selanjutnya buka folder project Quasar tersebut "belajar_quasar" dengan VS code, lalu buka terminal dan jalankan perintah berikut untuk melihat hasil aplikasi di browser.
quasar dev

Tunggu hingga proses kompilasi selesai, maka akan tampil server sudah running dan tampil di browser.

  1. Kemudian silakan pelajari dan pahami maksud struktur folder dan file yang ada dalam project Quasar.

Perhatian: Copy file dan folder satu project Quasar tersebut ke folder laporan src/13_quasar, lalu screenshot hasil di browser dan dibuat laporan. Kemudian push ke repo Anda masing-masing.

Melanjutkan praktikum sebelumnya, kita akan belajar tentang struktur project Quasar agar dapat membuat aplikasi pertama kita. Silakan lakukan langkah-langkah praktikum berikut ini.

  1. Pertama, sebelum kita memulai menggunakan Quasar Framework, sebaiknya kita memiliki pengetahuan dasar terkait ES6 dan cara kerja Vue.

Setelah Anda mempelajari fitur-fitur ES6 dan Vue.js, Anda akan lebih mudah memahami menggunakan komponen Quasar dan mengonfigurasinya. Beberapa hal dasar yang perlu diketahui adalah cara penggunaan komponen Quasar, properti yang ada di Vue, dan beberapa method dan event yang bisa digunakan.

Catatan: Silakan pelajari terlebih dahulu fitur-fitur ES6 dan Vue.js walaupun tidak secara menyeluruh dan mendalam, namun setidaknya Anda sudah membacanya sekilas sehingga memiliki pengetahuan dasar.

Vue Single File Components (SFC)

  1. Anda akan membangun aplikasi Quasar menggunakan file *.vue yang berisi beberapa bagian, yaitu:
  • template (HTML)
  • script (Javascript)
  • style (CSS/Stylus/SASS/SCSS/Less)

Semua bagian tersebut akan berada dalam satu file yang sama. Untuk template kode Quasar dapat dilihat pada kode berikut ini.

<template>
  <!-- you define your Vue template here -->
</template>

<script>
// This is where your Javascript goes
// to define your Vue component, which
// can be a Layout, a Page or your own
// component used throughout the app.

export default {
  //
}
</script>

<style>
/* This is where your CSS goes */
</style>
  1. Sekarang kita coba buat kode pertama dengan buka file src > pages > Index.vue
<template>
  <q-page class="flex flex-center">
    <h3>{{ hello }}</h3>
  </q-page>
</template>

<script>
export default {
  name: 'PageIndex',
  data() {
    return {
      hello: "Halo, Nama Saya Habibie dengan NIM 12345"
    }
  }
}
</script>

Perhatian: Silakan sesuaikan nama dan NIM dengan milik Anda.

  1. Save file Index.vue dan langsung lihat hasilnya di browser tanpa harus kita refresh, maka tampilan akan langsung update, karena Quasar memiliki fitur Hot Reloaded.

Perhatian: Copy file dan folder satu project Quasar tersebut ke folder laporan src/13_quasar, lalu screenshot hasil di browser dan dibuat laporan. Kemudian push ke repo Anda masing-masing.
